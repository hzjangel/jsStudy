# 参数的传递

# 参数默认值

在ES6中,可以设置参数的默认值,在ES5中不能设置参数的默认值

/**
 * num的n次幂
 * @param num 求幂的数
 * @param n 幂次数,默认是一次
 * @return {number}
 */
function pow(num, n = 1) {
    let sum = 1;
    for (let i = 0; i < n; i++) {
        sum = sum * num;
    }
    return sum;
}

console.log(pow(2, 3));
console.log(pow(2));

在设置参数的默认值的时候,需要把设置参数的默认值放在参数的最后,如果不放在参数的最后,那么这个参数就必须传值,如果需要使用默认值,那么就要显示的传递undefined

# 解构赋值和默认值的结合

函数中参数的传递可以使用赋值解构的方式进行传值

function register({email, password, pictureUrl="assets/image/defaultAvatar.png"}) {
    console.log(email, password, pictureUrl);
}

register({email: '123@qq.com', password: '123456'});
register({email: '123@qq.com', password: '123456', pictureUrl: 'upload.png'});
//不能什么都不传
register();

# 函数的length

函数的length的值为参数的个数,但是如果参数使用了默认值,这个参数不计在内

function fun(name, age) {
    console.log(name, age);
}
console.log(fun.length); //2
function fun1(name, age = 22) {
    console.log(name, age);
}
console.log(fun1.length); //1

# 作用域

给参数设置默认值的时候,会产生一个单独的作用域,初始化结束之后,这个作用域就会消失

let x = 4;
function fun(x, y = x) {
    console.log(y);
}
fun(); //undefined
fun(1); //1

在调用fun函数的时候,参数y设置了默认值,产生了一个单独的作用域,可以简答的理解为下面的代码

let x;
let y = x;

调用函数的时候并没哟传任何参数,所以x=undefined,在给y赋值的时候,先会在y所在的作用域查找,找到了直接返回,如果没有找到,在向上级作用域进行查找

如果这样写会直接报错

let x = 4;
function fun(y = x, x) {
    console.log(y);
}
fun(); //报错
fun(1); //报错
fun(1, 2); //不报错

对于参数里面可以这样理解

let y = x;
let x;

使用let声明变量x不能够先使用后定义,虽然在上级作用域定义了x,但是在本级作用域使用了let定义x,对于x来说并不会往上级进行查找,也不会做变量提升

# rest参数

用于获取函数的多余参数,语法为(...变量名)

例子

function puah(arr, ...item) {
    item.forEach(value => {
        arr.push(value)
    })
}

let arr = [];
puah(arr, 1, 2, 3, 4, 5);
console.log(arr); //[ 1, 2, 3, 4, 5 ]

rest参数只能在最后,在rest参数的后面不能够再有其他类型的参数

# 箭头函数

# 注意事项

  1. 箭头函数的this是固定的,是定义时所在的对象,并不是在使用时所在的对象

  2. 不能够当做构造函数,不可以使用new命令,因为箭头函数里面没有this

  3. 在箭头函数中是没有arguments对象

  4. 不能使用yield命令

  5. 在一行编写时，如果返回值是一个对象，那么需要将这个对象使用小括号包裹起来

this的指向例子

function Timer() {
    this.s1 = 0;
    this.s2 = 0;
    setInterval(() => {
        //这里的ths指向Timer
        this.s1++;
    }, 1000);

    setInterval(function () {
        //这里的this指向Windows
        this.s2++;
    }, 1000)
}

let timer = new Timer();
setTimeout(() => console.log(timer.s1), 3000);
setTimeout(() => console.log(timer.s2), 3000);

//一行编写
let fun = (name, age) => ({name: name, age: age});
//箭头函数的代码块多余一行,可以使用大括号包裹起来
let fun1 = (name, age => {
    return {name: name, age: age};
});

# 尾部调用

在函数的最后一步调用另外一个函数(在函数的return处调用另外一个函数,且不能有其他操作)

在使用为尾部调用的时候,外层函数会被销毁,不会保留下来

function f(x) {
    x = x + 2;
    return g(x);
}

function g(x) {
    return x * x;
}

console.log(f(2));

等价于下面代码

function f(x) {
    return g(x + 2);
}

function g(x) {
    return x * x;
}

console.log(f(2));

//等价于
function g(x) {
    return x * x;
}

console.log(g(2 + 2);

只有不再用到外层函数的内部变量，内层函数的调用帧才会取代外层函数的调用帧，否则就无法进行“尾调用优化”。

目前只有 Safari 浏览器支持尾调用优化，Chrome 和 Firefox 都不支持。

# 尾递归

一种特殊的尾调用,在最后一步调用自身

使用尾递归不存在栈溢出的情况,因为在调用自身的时候,已经将外层函数释放(销毁)了

function f(n, sum = 1) {
    if (n == 1) {
        return sum;
    }
    return f(n - 1, sum * n);
}

尾递归只有在es6中才有
